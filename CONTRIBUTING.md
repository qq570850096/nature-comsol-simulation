# Contributing

Contributions should keep the project narrow: COMSOL/FEA publication support,
not general simulation automation.

## Good Contributions

- New domain fragments with clear reviewer-risk logic.
- Better Methods, mesh-convergence, solver, validation, or reproducibility
  checklists.
- Realistic eval prompts based on simulation manuscript tasks.
- Clearer claim-downgrade language.
- Documentation that improves installation or project boundaries.

## Avoid

- Full solver execution pipelines.
- GUI automation as the main feature.
- Generic academic-writing templates unrelated to simulation evidence.
- Unverified claims, fabricated examples, or fake software/model metadata.

## Checklist

- Keep `SKILL.md` concise.
- Put detailed rules in `references/` or `static/fragments/`.
- Update `manifest.yaml` when adding a fragment.
- Add or update eval prompts.
- Run skill validation before submitting changes.
