# Artifact: Results

Write Results as claim-bounded interpretation of supplied model outputs.

Include:

- One sentence naming the model, study type, and output.
- A claim-evidence map before or after the prose when useful.
- Results prose that separates model prediction from mechanism or validation.
- Missing inputs and reviewer-risk flags.

Avoid:

- Methods overload in the Results paragraph.
- Unqualified real-world performance language.
- Numeric values not supplied by the user.

Useful structure:

1. Model context.
2. Primary simulated pattern.
3. Comparative or mechanistic interpretation, bounded by assumptions.
4. What additional validation or sensitivity analysis would elevate the claim.
