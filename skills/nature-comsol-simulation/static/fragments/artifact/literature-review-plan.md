# Artifact: Literature Review Plan

Use when the user asks for a review paper, related-work map, or literature
strategy around COMSOL/mechanical simulation.

Plan by argument, not by software name alone:

- Application domain: implant, material, fracture, thermal stress, dynamics,
  contact, FSI, etc.
- Model class: linear elastic, nonlinear, hyperelastic, fracture, contact,
  coupled multiphysics, dynamic.
- Evidence level: analytic benchmark, experimental validation, clinical/field
  validation, sensitivity/uncertainty, reproducibility.
- Claim type: mechanism, design optimization, prediction, workflow/method.
- Reporting quality: mesh convergence, boundary-condition transparency, solver
  settings, data/model availability.

Output should include search themes, comparison table columns, inclusion/exclusion
criteria, and a narrative thesis for the review.
