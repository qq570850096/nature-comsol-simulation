# Model Claim: Validation and Calibration

Use when claims depend on agreement with experiment, calibration, benchmark,
analytic solution, literature comparison, or uncertainty/error metrics.

Check:

- Calibration data and validation data should be separated when possible.
- State the measured quantity, experimental condition, sample size, error metric,
  and acceptance criterion.
- Verification asks whether the equations were solved correctly; validation asks
  whether the model represents the real system.
- Agreement in one output does not validate all outputs or operating regimes.
- If validation is planned but not done, write the claim as a model prediction.

Wording: "the model was calibrated to..." and "validated against..." only when
the corresponding evidence is supplied.
